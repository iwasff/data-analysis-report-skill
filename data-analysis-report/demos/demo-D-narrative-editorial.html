What

什么是
Report Skill？

不是一份报告，是一套生成报告的方法论——五个设计维度独立可配，Phase 0 逐轴问答后输出 Style Config，后续每份报告都保持一致风格。

Editorial 风格的核心是留白与字体，用排版本身传递信息层次，而不是依赖颜色对比或动效密度。

NARRATIVE MODE
Slide 叙事布局
全屏 scroll-snap，每页聚焦一个核心内容。Editorial 风格在 Narrative 模式下效果最佳——读者翻页而非滚动，信息接收更专注
EDITORIAL UI
字体驱动的设计
无玻璃毛玻璃，无渐变填充。卡片用细边框+左侧色条区分，排版比装饰更重要。适合内容密度低、观点鲜明的报告
02 / 07
Architecture

三层架构
让报告可信赖

计算与判断分离，Python 只做数学，Claude 只做语言——两层分工，根本上消除 AI 幻觉

LAYER 01
PRD 字段定义
product_requirements.md
每个指标的口径、分母、业务逻辑，用自然语言写成。整套体系的「单一真理源」——换人、换数据都不用重新对齐
LAYER 02
Python 纯计算
analyze.py → data.json
只做聚合、排序、计算，输出结构化 JSON，不包含任何自然语言判断。同样输入永远同样输出，可 diff，可验证
LAYER 03
Claude 报告生成
JSON → HTML Report
读取计算好的 JSON，生成洞察和行动建议。Claude 做它擅长的——语言与判断，把数学交给 Python
