Ice-Glass 与 Dark-Glass 的区别
两者都是 glass UI，但底色完全相反。Ice-Glass 是浅冰蓝渐变底 + 高透白玻璃（透明度 30%），内容在通透感中浮现；Dark-Glass 是近黑暗底 + 低透白玻璃（透明度 6%），更强调夜间科技感
glass-round 字体体系
Montserrat（西文数字，圆润几何感）+ 幼圆/YouYuan（中文，圆体）。两种字体气质高度统一，配合超大圆角卡片（24px），整体呈现柔和、通透、清爽的气质

Ice-Glass 图表配色
冷蓝绿色系 — 与背景渐变完全融合，图表本身像是从背景中「生长」出来的，而不是叠加上去的。严重度色也去饱和：ok 用 #7DA59C，bad 用 #9E7A7A，柔和而专业
卡片透明度对图表的影响
高透明度卡片（30%）在图表内部会透出背景渐变，需要在 ECharts 配置里把图表背景设为 transparent，让渐变自然透出，不用额外设置底色

Ice-Glass 地图配色 — 浅青绿渐变，与冰透背景融为一体，轻盈而专业
地图 · Ice-Glass 配色逻辑
空白区域用接近背景的浅冰蓝（#DEEAEB），数据区用从浅到深的蓝绿色阶，最高值区域用 #5C7E8C。整体像是在冰面上的水彩渲染，轻盈不沉重
什么场景适合 Ice-Glass 地图
明色系、易阅读、打印友好。适合全球市场报告、品牌分析报告、对外展示场景。相比 Dark-Glass 地图更柔和，不会给读者带来视觉压迫感
Architecture 三层解耦架构 核心设计
LAYER 01 · 定义层
PRD 字段定义
product_requirements.md
每个指标的口径、分母、业务逻辑，用自然语言写成。换人、换数据不需要重新对齐，是整套体系的「单一真理源」
可追溯 · 可传递
LAYER 02 · 计算层
Python 纯计算
analyze.py → data.json
只做聚合、排序、计算，输出结构化 JSON，绝不包含任何自然语言判断。同样输入永远同样输出——根本上消除 AI 幻觉
确定性 · 可验证
LAYER 03 · 洞察层
Claude 报告生成
JSON → HTML Report
读取计算好的 JSON，Claude 只做它擅长的——语言与判断。按 Style Config 调用对应维度代码库，生成完整单文件 HTML 报告
洞察 · 可视化
