# Report Components

## Analysis Mode Structure

```text
Cover
Overview KPIs
Section 1: main diagnostic view
Section 2-5: supporting analysis sections
Action plan or recommendations
Appendix / methodology when useful
```

## Narrative Mode Structure

Use 6-12 full-screen sections. Each section should have one message, one proof object, and one transition idea.

## Chart Patterns

- Ranking: horizontal bar with value labels.
- Mix/composition: stacked bar or compact pie only when categories are few.
- Trend: line chart with important inflection annotations.
- Association: heatmap table for Lift or co-occurrence.
- Risk: scatter plot with quadrant labels.

## Section Pattern

Each analysis section should contain:

1. Section title with business question.
2. Primary chart or table.
3. Two to four insight cards.
4. Short decision implication.
5. Data caveat when needed.

## Delivery Checklist

- HTML opens directly from disk.
- No missing external local files.
- No chart overlap or zero-height chart containers.
- KPI values match input JSON.
- Assumptions are visible.
